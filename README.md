# AiChatBot
Chatbot example using Claude managed agents, sessions, environment and memory stores written in React and .Net

![](/F4CIO.ChatBot.png)

A minimal, single-screen chatbot. A **React 19** front end talks to a **.NET 10** REST API, which
relays the conversation to **Claude** running as a server-managed **Managed Agent** (Anthropic runs
the agent loop in the background and persists context in a named **Memory store**). No database, no
authentication — you only fill in a few configuration values to get a working chat.

## Solution layout (n-tier monolith)

| Project | Type | Role |
|---|---|---|
| `F4CIO.AiChatBot.Common.Entities` | .NET 10 class library | POCOs passed between layers: `Configuration`, `Error`, `ChatRequest`, `ChatResponse`, `AppInfo`, `ErrorInfo` |
| `F4CIO.AiChatBot.Common` | .NET 10 class library | Central logging (`HandlerForLogging.AddLine`), error handling (`HandlerForErrors`), conversation log (`HandlerForConversationLog.AddMessage`), path resolution (`HandlerForPaths`) |
| `F4CIO.AiChatBot.Contracts` | .NET 10 class library | `IChatService` — the operation contract shared by the API, console, and (via OpenAPI) the web client |
| `F4CIO.AiChatBot.BusinessLogic` | .NET 10 class library | `ChatService` + `HandlerForClaude` — the Claude Managed Agents integration (official `Anthropic` NuGet) |
| `F4CIO.AiChatBot.UiApi` | ASP.NET Core 10 Web API | REST endpoints + Swagger/OpenAPI |
| `F4CIO.AiChatBot.UiConsole` | .NET 10 console | A REPL that drives the same `IChatService` — mirrors the web app |
| `F4CIO.AiChatBot.UiWeb` | React 19 + Vite + TypeScript | The single chat screen; typed API client generated by NSwag |

Build order is bottom-up: Entities → Common → Contracts → BusinessLogic → UiApi/UiConsole. The
solution file is `F4CIO.AiChatBot.sln`. (Requires Visual Studio 2022 **17.14+** or VS 2026 — the
first versions that support the .NET 10 SDK.)

## Prerequisites (development machine)

- **.NET 10 SDK** (10.0.3xx). Check with `dotnet --list-sdks`.
- **Node.js 20 or 22 LTS** + npm (for the React app only).

## Configuration — the one place you fill in

Edit the `Configuration` section in **`F4CIO.AiChatBot.UiApi/appsettings.json`** (and, if you use the
console, `F4CIO.AiChatBot.UiConsole/appsettings.json`):

```jsonc
"Configuration": {
  "AppTitle": "F4CIO AI Chatbot",          // shown in the UI header
  "ClaudeApiKey": "sk-ant-...",            // your Claude API key  <-- REQUIRED
  "ClaudeBaseUrl": "https://api.anthropic.com",
  "ClaudeAgentName": "F4CIO-Chatbot-Agent",   // found-or-created at startup
  "ClaudeMemoryStoreName": "F4CIO-Chatbot-Memory", // found-or-created; attached to each session
  "ClaudeEnvironmentName": "F4CIO-Chatbot-Env",    // found-or-created at startup
  "ClaudeModel": "claude-opus-4-8",
  "ApiBaseUrl": "http://localhost:5100"
},
"Cors": { "AllowedOrigins": [ "http://localhost:5101", "http://localhost:5102" ] }   // the SPA origin(s): dev + static
```

On first use the backend automatically creates the named Environment, Agent and Memory store if they
don't already exist, then attaches the Memory store to every chat session (this is what gives the bot
durable memory across sessions even without a database).

> The React app's API endpoint is **not** baked into the build — it's read at runtime from
> `F4CIO.AiChatBot.UiWeb/public/config.js` (`window.__API_BASE__`). See deployment below.

## Run locally

Use .bat files in /SourceCode root to build, run or publish. You should be able to build/run/setup it without any IDE!

Optionally use:
```powershell
# 1) Backend API  ->  http://localhost:5100  (Swagger UI at /swagger)
dotnet run --project F4CIO.AiChatBot.UiApi

# 2) Console client (same capability as the web UI)
dotnet run --project F4CIO.AiChatBot.UiConsole

# 3) Web UI (dev)  ->  http://localhost:5101
npm install --prefix F4CIO.AiChatBot.UiWeb
npm --prefix F4CIO.AiChatBot.UiWeb run dev
```

## Regenerating the React API client (NSwag)

The typed client in `F4CIO.AiChatBot.UiWeb/src/api/apiClient.ts` is generated from the API's OpenAPI
document. When you change the API, regenerate it in two clicks:

```powershell
# 1) Make sure the API is running (it serves /swagger/v1/swagger.json)
dotnet run --project F4CIO.AiChatBot.UiApi
# 2) In another terminal:
npm --prefix F4CIO.AiChatBot.UiWeb run gen:api
```

Configuration lives in `F4CIO.AiChatBot.UiWeb/nswag.json`.

## Where logs go

- **`F4CIO.AiChatBot.log`** — one per running host (the API or the console), written at that host's
  root folder. BusinessLogic is a library: its errors are logged by whichever host called it.
- **`Conversations/F4CIO.AiChatBot.Conversation<start>.txt`** — one file per conversation, next to the
  BusinessLogic assembly, named by the conversation start timestamp.
- Every caught error produces a `LogId` (a year→millisecond digit timestamp, e.g. `20260613004044015`)
  that is both logged and shown to the user, so a report can be traced to the exact log line.

# Deployment on Windows Server

> **Heads-up React app is **built on your dev machine** and only the static output is copied to
> the server. Node is never installed on the server.

## 1. One-time server prerequisites

1. (optional) Ensure **ESU** is active on the box.
2. (optional or if you get errors) **Enable TLS 1.2** (2012 R2's defaults can't reach `api.anthropic.com`). Add these registry keys and
   reboot:
   ```
   [HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.2\Server]  "Enabled"=dword:00000001 "DisabledByDefault"=dword:00000000
   [HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.2\Client]  "Enabled"=dword:00000001 "DisabledByDefault"=dword:00000000
   [HKLM\SOFTWARE\Microsoft\.NETFramework\v4.0.30319]               "SchUseStrongCrypto"=dword:00000001 "SystemDefaultTlsVersions"=dword:00000001
   [HKLM\SOFTWARE\Wow6432Node\Microsoft\.NETFramework\v4.0.30319]   "SchUseStrongCrypto"=dword:00000001 "SystemDefaultTlsVersions"=dword:00000001
   ```
3. Install **IIS** with the **URL Rewrite** module (needed for the SPA). If you host the API
   framework-dependent, also install the **.NET 10 Hosting Bundle**; if you publish self-contained
   (recommended below), IIS alone is enough.

## 2. Backend (UiApi)

Publish on your dev machine — **self-contained** so the server doesn't need a separate .NET runtime.
Run **`publish-api.bat`** (or in VS: UiApi → Publish → **FolderProfile**). Output goes to
`..\Published\F4CIO.AiChatBot.UiApi` (a sibling of `SourceCode`). Manual equivalent:

```powershell
dotnet publish F4CIO.AiChatBot.UiApi -c Release -r win-x64 --self-contained true -o ..\Published\F4CIO.AiChatBot.UiApi
```

On the server:
- Copy `Published\F4CIO.AiChatBot.UiApi` to e.g. `C:\inetpub\F4CIO.AiChatBot.Api`.
- Create an IIS **site** (or application) pointing at it, with an **App Pool set to "No Managed Code"**.
- The published `web.config` uses the ASP.NET Core Module to launch the app. Set the listening
  URL/port as needed.
- Put your real values in `appsettings.json` (or `appsettings.Production.json`) — especially
  `ClaudeApiKey` — and set `Cors.AllowedOrigins` to the **front-end site URL**.

## 3. Front end (UiWeb) — build off-server, host static

1. **Build on your dev machine** (Node 20/22): run **`publish-web.bat`**. It builds the app and copies
   the static site to `..\Published\F4CIO.AiChatBot.UiWeb` (sibling of `SourceCode`), including
   `web.config`, `config.js`, and `run-uiweb.bat`. (Manual equivalent: `npm ci --prefix
   F4CIO.AiChatBot.UiWeb` then `npm --prefix F4CIO.AiChatBot.UiWeb run build`, which emits `dist/`.)
2. **Copy `Published\F4CIO.AiChatBot.UiWeb`** to the server, e.g. `C:\inetpub\F4CIO.AiChatBot.Web`.
3. **Create a static IIS site** pointing at that folder. The included **`web.config`** already
   provides the SPA fallback rewrite and the MIME types older IIS may lack (it requires the URL
   Rewrite module from step 1). Or, for a quick non-IIS run, double-click **`run-uiweb.bat`** in that
   folder (a Node-free PowerShell static server with SPA fallback).
4. **Point the SPA at the API without rebuilding** — edit `config.js` on the server:
   ```js
   window.__API_BASE__ = "https://your-server/api-base";   // the UiApi site's URL
   ```
   This is the only file you touch to repoint the front end; no rebuild required.
5. Bind an HTTPS certificate and make sure the API's `Cors.AllowedOrigins` includes this site's URL.

## 4. Updating the API client after backend changes

On the dev machine: start the API → `npm --prefix F4CIO.AiChatBot.UiWeb run gen:api` → `npm --prefix
F4CIO.AiChatBot.UiWeb run build` → copy the new `dist/` to the server. (You normally don't need to edit
`config.js` again — it persists on the server.)

## Notes

- **Managed Agents is a beta API.** The integration is isolated in `HandlerForClaude`
  (BusinessLogic), so it's the only place to touch if the SDK surface changes.
- Default model is `claude-opus-4-8`; change `ClaudeModel` in configuration to use another.
- There is intentionally **no database and no authentication** — conversation continuity within a run
  is kept in memory in the API process, while long-term memory lives in the Claude Memory store.
